# Deductrium 攻略第一章 命题逻辑

## 初识推理层

- 【解锁了成就】进入游戏后，向画面上方前进，解锁进度层
- 【吃素啦】解锁进度层后，在作者信息处左拐，再左拐，获取 1μg 推理素。这里有一条岔路显示“前面的道路以后再来探索吧”，需要大后期才能开启
- 回到游戏开始处向另一侧前进，解锁推理层。开局有三个推理规则
>  mp: ($0 → $1), $0 ⊢ $1
>  a1: ⊢ $0 → $1 → $0
>  a2: ⊢ ($0 → $1 → $2) → ($0 → $1) → ($0 → $2)

  在这里箭头运算采用右结合性，即 a → b → c 相当于 a → (b → c)。

  mp 是假言推理，它是说如果 a → b 为真，且 a 为真，那么 b 为真。

  a1 公理是说如果 a 为真，那么 b → a 也为真。换言之真命题可以任意添加前提条件。

  a2 公理是说如果 ① a → b → c 为真，且 ② a → b 为真，那么 a → c
  为真。a2 公理是一种分配律，已知 a → (b → c)，它将前提 a 分配到命题 b → c 的两端，即
  (a → b) → (a → c)。
  > 可以这样理解 a2：要证 a → c 为真，先假设前提 a 为真，利用已知条件① 可知 b
  → c 为真，利用② 可知 b 为真。综合这两条得到 c 为真。于是证明了 a → c 为真。
  >
  > 当然在游戏中，a2 是已知的公理，我们不必证明 a2，而是要用它证明其它的推理规则。

## 我推出我

- 接下来我们解决第一个谜题 #p: a→a→a。在推理层输入下面井号右边的命令即可
> p0: a→a→a  # a1 a a

  【我推出我】通过这扇门再向前走，有推出 a→a 的提示。

>  p1: a→(a→a)→a                  # a1 a a>a
>  p2: (a→(a→a)→a)→(a→a→a)→(a→a)  # a2 a a>a a
>  p3: (a→a→a)→(a→a)              # mp 2 1
>  p4: a→a                        # mp 3 0
  
  类似可以证明 b→b 和 (a→b)→(a→b)
- 可以连续输入多条命令，例如一次性粘贴下面所有命令可以直接证明 a → a：
  ```deduct
  a1 a a
  a1 a a>a
  a2 a a>a a
  mp 2 1
  mp 3 0
  ```
  把换行换成空格也是可以的。以后我们都用这种简短风格书写证明，不再把每一步推理结果写出。
- 【第一次消费】【收费站拆除】通过门 a→a 以后，如果刚好拥有 2μg 推理素，可以直接解锁拆门技能。
- 否则可以先到救济站领取 5μg 推理素。救济站门口写着“被耗光了推理素但不想重置游戏？请跟我来！”但救济站里的路有亿点长就是了（来回80格）。
- 【你推出你，他推出他】花 1μg 解锁录制宏 (m)，然后录制第一条推理规则：<br>
  箭头的同一性（identity）si: $0 → $0。
  方法如下：先在定理列表中推理得到 $0 → $0, 然后输入命令：
  ```deduct
  m si
  ```
  有了这条推理规则就可以打开 #d: $0 → $0 这扇门
- 【If I were..】解锁假设 (hyp)。
  推理过程中可以随时添加假设到定理列表上方。可以拖拽调整假设的顺序。如果假设被拖拽到定理列表的下方，该假设会被移除。

## 元定理 ( c ) (>)

- 箭头的传递性 (transitivity) st: ($0 → $1), ($1 → $2) ⊢ $0 → $2
  ```deduct
  hyp $0>$1 $1>$2
  a1 $1>$2 $0
  a2 $0 $1 $2
  mp 2 1
  mp 3 4
  mp 5 0
  ```
- 解锁条件演绎元定理：要证 $0→$1→$2, $0→$1→$2→$3 ⊢ $0→$1→$3
  ```deduct
  hyp $0>$1>$2 $0>$1>$2>$3
  a2 $0 $1>$2 $1>$3
  a2 $1 $2 $3
  st 1 3
  mp 2 4
  mp 5 0
  ```
- 条件演绎元定理通过命令前缀 c 来使用。它的作用是把推理规则 $0 ⊢ $1 变成 $2→$0 ⊢ $2→$1，也就是加上相同的前提条件。比如 cmp 就表示 $2→$0→$1, $2→$0 ⊢ $2→$1 。我们利用条件演绎元定理证明下面的推理规则：<br>
  条件的交换律 (condition swap) scs: $0→$1→$2 ⊢ $1→$0→$2
  ```deduct
  hyp $0>$1>$2
  a1 $0>$1>$2 $1
  mp 1 0
  a1 $1 $0
  ccmp 2 3
  ```
- 【会跑的⊢】解锁演绎元定理：要证 ($0→$1→$2)→($0→$1→$2→$3)→($0→$1→$3)<br>
  这基本就是把条件演绎元定理前的关卡再推一遍，但这次没有假设，我们需要一个无假设版本的 st 规则:
  s>>t: ($0→$1)→($1→$2)→($0→$2)
  ```deduct
  a1 $1>$2 $0
  a2 $0 $1 $2
  st 0 1 # 先证 ($1→$2)→($0→$1)→($0→$2)
  scs 2  # 再用 scs 交换条件
  ```
  
  回头解决谜题 ($0→$1→$2)→($0→$1→$2→$3)→($0→$1→$3)：
  ```deduct
  a2 $0 $1>$2 $1>$3
  a2 $1 $2 $3
  s>>t $0 $1>$2>$3 ($1>$2)>($1>$3)
  scs 2
  mp 3 1
  st 4 0
  scs 5
  ```
- 演绎元定理通过命令前缀 > 来使用，它将推理规则 $0 ⊢ $1 变成 $0 → $1。这相当于把条件右移至 ⊢ 的右边。
  有了条件演绎元定理和演绎元定理，回头再看两个谜题就非常简单了: 比如已有一个推理规则
  smp: $0, $0→$1 ⊢ $1, 那么下面一条命令就能立即证得谜题答案
  ```deduct
  >>ccsmp $0 $1 $2 $3
  ```
  现在我们先前录制的宏 s>>t 也可以删除了，因为它完全等价于 >>st。
  让我们保留 si, st, scs, smp 4 个宏继续前进。

> 推理规则列表可以拖动排序

- 在游戏中行走步数超过 256 步即可拆除健身俱乐部的门，并进入一个迷宫寻找指南针。
- 防迷路小技巧：你也许听说过，永远扶着右手边的墙前进，就能走出迷宫。但如果忘记了前进的方向呢？因此一个好习惯是，永远将画面上方设定为前进的方向。由于游戏是在双曲平面上进行，走一段路后前进方向会发生偏移，此时可以拖动圆盘外侧旋转方向。

## 否定推理

- 【敢于说不】解锁否定公理 a3
  ```deduct
  a3: (¬$0 → ¬$1) → ($1 → $0)
  ```
  a3 是说要证命题 b → a，只需证它的逆否命题 ¬a → ¬b
- 解锁 a3 后，找到一个格子上面写着“提示：解锁符号~后才能开这些门”。
  有了 st 规则后，结合游戏提示，这些谜题并不复杂：
  ```deduct
  a1 ~~a ~~~~a
  a3 ~~~a ~a
  st 0 1
  a3 a ~~a
  st 2 3
  a2 ~~a ~~a a
  mp 5 4
  si ~~a
  mp 6 7
  ```
  至此健身俱乐部这一侧的道路探索完毕。
- 【负负得正】回到解锁 a3 的地方，现在我们可以打开门 #p: ¬¬a→a。
  这是一条重要规则，建议录制为双重否定消去律（negative elimination）sne: ¬¬$0 → $0。
- 类似有双重否定增加律（negative incrementation）sni: $0 → ¬¬$0
  ```deduct
  a3 ~~$0 $0
  sne ~$0
  mp 0 1
  ```
- 【拆除门#K】a3 的逆命题 sa30: $0→$1 ⊢ ¬$1→¬$0
  ```deduct
  hyp $0>$1
  a3 ~$0 ~$1
  sne $0
  sni $1
  st 2 0
  st 4 3
  mp 1 5
  ```
  下面两条规则证明类似： sa31: ¬$0→$1 ⊢ ¬$1→$0 和 sa32: $0→¬$1 ⊢ $1→¬$0
- 【否定爆炸】sm: $0 ⊢ ¬$0→$1
  ```deduct
  hyp $0 ~$0
  a1 $0 ~$1
  mp 2 0
  sa31 3
  mp 4 1
  ```
- 相同条件消除 sce: $0→$0→$1 ⊢ $0→$1。提示：使用 a2，在证双重否定消去时我们已经见过这个推理。

  至此 a3 附近的门清理完毕。
- 我们回到解锁拆门技能的地方，利用已有的否定爆炸打开这些门。

## 反证法

- 拆除门 #<>(↔)：`s>TF: $0,¬$1 ⊢ ¬($0→$1)`。
  这个谜题是说，要否定命题 $0→$1，只需它的前提 $0 为真，且结论 $1 为假。
  直接证明它比较困难。我们可以用反证法，先假设结论不成立（即 $0→$1），然后推出矛盾。

  引理 `tmp1: $0, ¬$1, $0→$1 ⊢ $1`
  ```deduct
  hyp $0 ~$1 $0>$1
  mp 2 0
  ```

  用引理证明 `s>TF`：
  ```deduct
  hyp $0 ~$1
  >tmp1 0 1
  sa30 2
  mp 3 1
  ```
  在把它录制下来之前，我们先执行命令 `inln 2` 把引理 tmp1 展开。然后 `del tmp1` 删除引理。
  现在可以执行 `m s>TF` 录制规则。这种“证明引理→展开引理→删除引理”的技巧，今后还会多次使用。

- `s>TF` 的直接证明。可以先证 $0→($0→$1)→$1，然后应用 sa30 将后面两项改为逆否：
  ```deduct
  hyp $0 ~$1
  >>smp $0 $1
  csa30 2
  mp 3 0
  mp 4 1
  ```

- `smn: (¬$0→$0)→$0`。也是先证逆否 `¬$0→¬(¬$0→$0)`：
  ```deduct
  >>s>TF ~$0 $0
  sce 0
  a3 $0 ~$0>$0
  mp 2 1
  ```
  请注意这条规则与否定爆炸 sm 的区别：sm 说如果 $0 和 ¬$0 **同时为真**，那么任意命题 $1 都为真。但 smn 说 ¬$0→$0，只能推出 $0 为假。
- `sm&: $0,$1 ⊢ ¬($0→¬$1)`。这是 `s>TF` 的推论。
  ```deduct
  hyp $0 $1
  sni $1
  mp 2 1
  s>TF 0 3
  ```
- `sm&1: ¬($0→$1)→$0`。它和下一条规则都是 a3 系列规则的应用
  ```deduct
  >>sm $0 $1
  scs 0
  sa31 1
  ```
- `sm&2: ¬($0→$1)→¬$1`
  ```deduct
  a1 $1 $0
  sa30 0
  ```
- 【逻辑门（与/或）】完成以上谜题后可以解锁与 (∧) (&) 或 (∨) (|) 两个符号

【命题逻辑大礼包】到这里相信你已拥有 101μg 推理素了，直接沿着指南针回到出生点，解锁属于你的大礼包吧。

 ## 新的元定理 (<) (\:)
- 【我推出你，你推出我】拆除门 #<>(↔) 之后，回到我们的起点，可以解锁<>(互推)
> d<>1: (($0 <> $1) → \~(($0 → $1) → \~($1 → $0)))
> d<>2: (\~(($0 → $1) → \~($1 → $0)) → ($0 <> $1))
- 与此同时 我们会发现有一个49μg的门
  里面有两个元定理 逆演绎元定理(midt)和组合元定理(mcmt)
  逆演绎元定理主要就是对应的定理与mp的结合 通过前缀<来使用
  而组合元定理值得好好来说一说
  组合元定理会把两个带条件(0个条件也行)的定理a与b自动匹配组合成一条规则 ":a,b" (通过前缀:来使用)
  比如 ":a2,<a1"将自动得到规则($3 → (($0 → ($1 → $2)) → (($0 → $1) → ($0 → $2)))) (其实就是ca2)
  以及 ":a1,mp"可得到<a1
- 事实上组合元定理十分强大,接下来为大家带来组合元定理的进阶用法
  1. 组合元定理的嵌套 
  组合元定理可以嵌套使用   如::a,b,c 等同于令m=:a,b   则::a,b,c=:m,c
  第二个条件同理
  2. 多规则条件的组合与#的使用
  第二个参数规则可以有多个条件
  如:a :b,c 表示先执行规则a和b,然后把它们的结果作为c的条件来执行c
  条件之间用:分隔 条件与结果之间用,分隔
  "#"放入为默认条件,条件不够默认为# (也就是#在这里作为占位符)
  可以输入以下规则体会一下：
  :<a1,cst
  :<a1:#,cst
  :#:<a1,cst
- 如下是一些使用组合元定理的实例
  si=:a1 :a1,<<a2
  st=:<a1,<a2 (条件是反的)
  scs=:a1 :<a2,st
  sni=:sne,<a3
  sm=:<sni,:<a1,<a3
  sm&1=:::csni:>>sm,cst:sni,st,<a3
  sm&2=:::sne:a1,st:sni,st,<a3
- 甚至可以用组合元定理模拟其他元定理
   ca1=:a1,<a1
   <a1=:a1,mp
   cmp=<<a2 (这个和组合元定理无关)
   csi=:a1,scs
   c<a2=::a2,<a1:#,cmp
- 组合元定理可以把多行的证明过程压缩到寥寥几行 甚至一行 但它最大的用处和强力性要在一阶逻辑才能得到良好的体现
- 如下是群友ℵ₁提供的部分组合元定理
  sne = ::a2:::a1:a3,st:a3,st,mp:si,mp
  st = <<:>:<a1,<a2,scs (这个是正的)
  scs(不依赖st) =::#,<a1:a1,ccmp
  sa30 = <::::sne,<a1:si,cst::sni,<a1,cst:a3,st
  sa31 = <::si::sni,<a1,cst:a3,st
  sa32 = <:csni:>sa30,cst
  smn = :::::csi:a1,ccmp,csa30,<a2:si,mp,<a3
  sm& = ::a3::sne:::a2:si,mp::a1:#,mp,mp,st,mp:#,mp
  s>TF = :::si:csni,cst,sa30::#:#,sm&,mp
  
 ## 等价 
- 让我们回到正题 刚刚我们解锁了d<>1和d<>2 现在回到拆门技能那里 
- 这里分为3条路,我们先走互推那条 
- s<>: ($0→$1),($1→$0) ⊢($0<>$1)
  ```deduct
  hyp ($0>$1) ($1>$0)
  sm& 0 1
  <d<>2 2
  ```
  s<>i: ($0<>$0) 十分简单,自证不难
- s<>1: ($0<>$1) ⊢ ($0→$1) 
  ```deduct
  hyp ($0<>$1)
  <d<>1 0
  <sm&1 1
  ```
  将<sm&1替换为<sm&2可证得s<>2: ($0<>$1) ⊢($1→$0)
- 在有了s<>,s<>1,s<>2后s<>s:($0<>$1) ⊢($1<>$0)和s<>t: ($0<>$1),($1<>$2) ⊢($0<>$2)的证明也不成问题
- sn:($0<>\~\~$0)也十分的简单
- 【拆除门#L】($0<>$1) ⊢(\~\~$0→$2)<>($1→$2) 推理的提示 
  运用演绎元定理 sni sne s<> s<>1 s<>2
  注：开#L需要130μg推理素,但实际消耗只有30μg

## 完备性元定理(mcpt)

- 从出生点出发，经过解锁推理层的格子后继续前进，完备性元定理位于右拐的一条小巷中，需要先解锁门 #K
- 在解锁完备性元定理之前 我们还需要推两个命题
- sm1: (\~$0→\~$1),(\~$0→$1) ⊢ $0
  ```deduct
  hyp (~$0>~$1) (~$0>$1)
  sa31 1
  st 0 2
  <smn 3
  ```
- sm2: ($0→$1),(\~$0→$1) ⊢ $1
  ```deduct
  hyp ($0>$1) (~$0>$1)
  sa30 0
  sa31 1
  sm1 2 3
  ```
- 终于,我们解锁了完备性元定理,它可以自动证明所有正确的命题逻辑的命题 你可以向它输入一个命题 如果其正确 则会加入到你的推理规则里
## 且与或
- 接下来我们走且(&)(∧)的路线
  在此之前 我们会获得两个定义:
> d&:(($0 & $1) <> \~($0 → \~$1))
> d|:(($0 | $1) <> (\~$0 > $1))
- s&: $0,$1 ⊢($0&$1)
  ```deduct
  hyp $0 $1
  sm& 0 1
  d& $0 $1
  <s<>2 3 2
  ```
- s&1: ($0&$1) ⊢ $0
  ```deduct
  hyp $0&$1
  d& $0 $1
  <s<>1 0 1
  <sm&1 2
  ```
  将<sm&1替换为<sm&2可证得s&2: ($0&$1) ⊢ $1
- 有了s& s&1 s&2后 证明s&s:($0&$1)⊢($1&$0) 和 s&a: ($0&$1)&$2  ⊢ $0&($1&$2)将相当简单
- 接下来是s&n1: \~$0 ⊢ \~($0&$1)
  ```deduct
  hyp ~$0
  >s&1 $0 $1
  <sa30 1 0
  ```
  将>s&1换为>s&2可证得s&n2: \~$1 ⊢ \~($0&$1)
- 下一个是或(|)(∨)的路线:
- s|1: $0 ⊢ ($0|$1)
  ```deduct
  hyp $0
  sm 0
  d| $0 $1
  <s<>2 2 1
  ```
- s|2: $1 ⊢ ($0|$1) 
  ```deduct
  hyp $1
  <a1 0 ~$0
  d| $0 $1
  <s<>2 2 1
  ```
- s|i: ($0|$0) ⊢ $0
  ```deduct
  hyp $0|$0
  d| $0 $0
  <s<>1 1 0
  <smn 2
  ```
- s|s ($0|$1) ⊢ ($1|$0)
  ```deduct
  hyp $0|$1
  d| $0 $1
  <s<>1 1 0
  sa31 2
  d| $1 $0
  <s<>2 4 3
  ```
- s|a: ($0|$1)|$2 ⊢ $0|($1|$2)
  ```deduct
  hyp ($0|$1)|$2
  d| $0 $1
  d| $0|$1 $2
  <s<>1 2 0
  s<>1 1
  sa30 4
  st 5 3
  <a1 6 ~$1
  <a1 7 ~$0
  >>sm& ~$0 ~$1
  cccsni ~$0 ~$1 ~$0>$1 $1
  ```
