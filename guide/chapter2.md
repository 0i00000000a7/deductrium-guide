# Deductrium 攻略第二章 一阶逻辑
## 开始
- 欢迎来到一阶逻辑的世界
- 这里默认你已经获得命题逻辑大礼包
- 在进入一阶逻辑后 我们会发现元规则列表增加了mq 规则列表增加了a4 a5 a6
  > mq: (##axiom(⊢$0))⊢M(##axiom(⊢∀$1:$0))
  > a4: (∀$0: $1) > #rp($1, $0, $2)
  > a5: (∀$0: ($1 > $2)) > ((∀$0: $1) > (∀$0: $2))
  > a6: #nf($1, $0) > (∀$0: #nf($1, $0))
  
- mq的解析：
  使用条件:
  1. 公理(如a1,a2,d<>1,d&等)
  2. 没有前提(这一条导致不能在只有mq的时候直接写vmp)
  通过前缀v来使用mq
  使用例子：如va1：
  ∀$2:($0→($1→$0)) 
- a4的解析
  #rp是replace(替换)的缩写
  #rp($1,$0,$2)的结果为将$1中自由出现的$0全部替换为$2所得到的东西(自由出现的定义请看a6)
  a4执行替换前会验证能否替换
  a4使用例子：sVe: (∀$0:$1) ⊢$1
  
  ```deduct
  hyp V$0:$1
  <a4 0 $0
  ```
  (因为把$0换成$0等于啥都没换)
- a5的解析
  a5就是任意(∀)的分配律
  使用例子：(∀$2:$0)→(∀$2:$1→$0)
  ```deduct
  va1 $2 $0 $1
  a5 $2 $0 $1>$0
  mp 1 0
  ```
  
  实际上这就是>v<a1
  如果你有组合元定理的话,将它与a5结合使用 可以帮助你更快的解锁后续内容
  如v<a1=<:va1,<a5
  v<a2=<:va2,<a5
  vcmp=v<<a2=<<::va2,<a5,c<a5
  vmp=<<a5
  
- a6的解析
  #nf是not free的缩写
  当$0不在$1中自由出现时#nf($1,$0)的结果为$1,$0如果自由出现则会报错
  自由出现的定义：
  1. 如果$0中不含一阶逻辑量词(∀,∃...)则自由出现指$0中含有$1
  2. 如果$0含有一阶逻辑量词,则设$0为∀(∃同理)x:$2           (1)若x为$1,则$1不在$0中自由出现 若$0中有x,这时我们说x被量词约束 即#nf(∀$1:$0,$1)=$0   (2)若x不为$1,则#nf(∀x:$2,$1)=∀x:#nf($2,$1)(x不为$1) 
  由于$开头的公式和项可换成任意值 无法确定 #nf将会保留
  使用例子 (∀x:$1)→(∀x:(∀x:$1)) 
  ```deduct
  a6 Vx:$1 x
  ```
## 量词,公式与项的介绍
- ∀(V)(任意)与∃(E)(存在)叫做量词 量词后紧跟的叫变量 冒号(:)后的公式叫该量词的管辖范围 整个∀(∃)$0:$1叫公式
- 注意：变量都是“项”,无法谈论真假 只有公式才能谈论真假 纯命题逻辑都是公式 可以谈论真假
- 谓词属于(∈)(@)与相等(=)只能连接项 连接后的整体是公式 如x@y与x=y都是公式 可以谈论真假
- 所有命题逻辑符号(>,<>,~,&,|)都只能连接公式,不能直接连接项 如输入"Vx:x>x"系统会报错 因为x在V后是变量 属于项 而>不能连接项 命题逻辑符号连接的整体也是公式
## 等词公理a7与a8
- a7解锁位置 (Vx:x@y)>(Vx:x@y)
- a8解锁位置 (V$0:$1)>$1
>a7: ($0=$0)
>a8: ($0 = $1) > ($2 > #rp($2, $0, $1, $3))
- a7解析：就是一个东西和它本身相等
- a8解析：
  有4个参数的#rp中前3个与普通#rp一致 第4个参数可输入数字(整数)来指定仅替换第几个出现的变量 0指全部替换 +n指从左到右数第n个 -n指从右往左数第n个
  (第4个参数可以别的东西 但没有什么意义)
  a8说的是如果a=b 那么将c中的所有或部分a替换成b依旧成立
  a8的使用提示：
   若想把$ a=$ b中的$ b替换为$ c 写#rp($ a=$ b,$ b,$ c,0) 结果会得到#rp($ a,$ b,$ c)=$ c 这是因为$ a中可能含有$ b 不知道是否替换 写#rp($ a=$ b,$ b,$ c,-1)即可解决问题 
-  a8的使用实例: 
   s=s: ($0=$1) ⊢ ($1=$0)
   
   ```deduct
   hyp $0=$1
   a7 $0
   <<a8 0 1 1
   ```
   s=t：($0=$1),($1=$2) ⊢($0=$2)
   ```deduct
   hyp $0=$1 $1=$2
   <<a8 1 0 -1
   ```
- 假如你现在有100μg推理素 那么你现在就可以解锁PA ZFC 和类型论了 (分别是第3 4 6章的内容)
## 重走万里长路(解锁门#l1与mcvt)
- mq只能给公理加上∀ 而a6加∀需要nf 但我们的命题逻辑的命题不是公理又没有nf 那该怎么办呢？
- 事实上 有一个东西能满足我们的要求 但在此之前 我们必须手动为命题逻辑的命题加∀
- 提示：有组合元定理会大幅简化流程
- 如： v.i=:va1:va1,<<::va2,<a5,c<a5
- v.t=<<:>::<:va1,<a5,<:va2,<a5,<a5,.cs
- v.cs=:va1:v<a2,v.t
- 相信你有了这些例子之后一定理解如何了手动+∀
- 【解锁门#I1】:该门需要.<> v.<>1和v.<>2 (该门实际上就是<>.rV)
- 【解锁mcvt】:该门需要.& v.& .&1 .&2 v.&1 v.&
  (该门实际就是定理V&)
  V&1:(∀$0:($1&$2)) ⊢ ((∀$0:$1)&(∀$0:$2))
  V&2:((∀$0:$1)&(∀$0:$2)) ⊢(∀$0:($1&$2))
  这两个命题是中间过程 建议录制并保留
  mcvt (条件概括元定理)可以把条件与结论加上相同的量词(∀) 通过前缀v来使用 它是mq的升级版
  首先它不受公理的限制 其次也不用管有几个条件 我们现在就可以直接生成vmp v.i v.t v.&... 无须再推导或用繁琐的组合元定理生成了

## 存在(∃)(E)
- 解锁定义dE:
> dE: ((∃$0: $1) <> ~(∀$0: ~$1))
- dE的解析：
  dE是说如果存在$0使$1成立,那么就不是所有的$0都不能使$1成立
- dE的推论
> nEVn:(~(E$0: $1) <> (V$0: ~$1))
> nVEn:(~(V$0: $1) <> (E$0: ~$1))
> nVVn:((E$2: (E$0: $1)) <> ~(V$2: (V$0: ~$1)))
> nEn:((V$0: $1) <> ~(E$0: ~$1))
- 下面为大家带来nVVn的推导过程

  ```deduct
  v.nEVn $2 $0 $1
  .<>rV 0
  .<>rn 1
  dE  $2 E$0:$1
  .<>t 3 2
  ```

## ∃相关的基本定理

  .Erp: (#rp($1,$0,$2))→(∃$0:$1)
- 接下来是推理过程:

  hyp $1
  >.Ve $0 ~$1

  a4 $0 ~#rp(#nf($1,$0),$2,$0) $2
